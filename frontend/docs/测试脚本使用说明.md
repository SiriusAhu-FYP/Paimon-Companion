# Live2D API 测试脚本 - 快速参考

## 🚀 快速开始

### 1. 启动开发服务器
```bash
pnpm dev
```

### 2. 运行测试脚本

> 所有脚本支持 `--port/-p` 与 `--api-base` 可选参数，可方便地切换目标：
> - `--port 8080`：连接到指定端口
> - `--api-base http://127.0.0.1:9000`：自定义完整 API 地址
>
> 也可以设置环境变量 `LIVE2D_API_BASE` 或 `FRONTEND_PORT`/`VITE_PORT` 来改变默认地址。
```bash
# 查看所有模型
node test_models.js

# 执行第3个动作（索引从0开始）
node test_motion.js 3

# 指定模型执行动作
node test_motion.js 3 Haru
```

---

## 📋 测试脚本列表

### `test_motion.js` - 动作测试 ⭐
检测当前角色的动作列表，然后执行第n个动作

```bash
# 仅列出所有动作
node test_motion.js

# 执行第3个动作（从0开始数）
node test_motion.js 3

# 执行指定模型的第3个动作
node test_motion.js 3 Haru
```

**输出内容：**
- ✅ 所有动作的列表及编号
- ✅ 每个动作的分组、文件、声音信息
- ✅ 执行指定动作的结果

---

### `test_expression.js` - 表情测试
检测当前角色的表情列表，然后执行第n个表情

```bash
# 仅列出所有表情
node test_expression.js

# 执行第2个表情（从0开始数）
node test_expression.js 2

# 执行指定模型的第2个表情
node test_expression.js 2 Haru
```

---

### `test_sound.js` - 声音测试
检测当前角色的声音列表，然后播放第n个声音

```bash
# 仅列出所有声音
node test_sound.js

# 播放第1个声音（从0开始数）
node test_sound.js 1

# 播放指定模型的第1个声音
node test_sound.js 1 Haru
```

---

### `test_models.js` - 模型信息
查看所有模型及其详细信息

```bash
# 列出所有模型概览
node test_models.js

# 显示指定模型的详细信息
node test_models.js Haru
```

---

### `test_api.js` - API集成测试
测试所有API端点的功能

```bash
# 运行完整测试
node test_api.js

# 仅列出资源
node test_api.js list

# 仅测试动作
node test_api.js motion

# 仅测试表情
node test_api.js expression

# 仅测试声音
node test_api.js sound
```

## 💡 使用示例

### 示例 1：查看并执行动作
```bash
# 步骤1：查看Haru模型有哪些动作
node test_motion.js

# 输出类似：
# [0] Idle_0
# [1] TapBody_0
# [2] TapBody_1
# [3] TapBody_2
# ...

# 步骤2：执行第3个动作
node test_motion.js 3

# 输出：
# 🎬 正在执行动作 [3]: TapBody_2
# ✅ 动作执行成功！
```

### 示例 2：切换模型测试
```bash
# 查看所有可用模型
node test_models.js

# 输出：
# [0] Haru
# [1] Hiyori
# [2] Mao
# ...

# 测试Hiyori的第5个动作
node test_motion.js 5 Hiyori

# 测试Mao的第1个表情
node test_expression.js 1 Mao
```

### 示例 3：完整测试流程
```bash
# 1. 查看模型详情
node test_models.js Haru

# 2. 测试动作
node test_motion.js 3 Haru

# 3. 测试表情
node test_expression.js 1 Haru

# 4. 测试声音
node test_sound.js 0 Haru

# 5. 运行完整测试
node test_api.js
```

## ⚠️ 注意事项

1. **必须先启动服务器**
   ```bash
   pnpm dev
   ```
   服务器地址：http://localhost:7788

2. **索引从0开始**
   - 第1个动作的索引是 0
   - 第2个动作的索引是 1
   - 第3个动作的索引是 2

3. **在浏览器中查看效果**
   - 测试脚本只发送命令
   - 实际效果需要在浏览器中查看
   - 打开：http://localhost:7788

4. **模型名称区分大小写**
   - ✅ 正确：`Haru`
   - ❌ 错误：`haru`

## 🔧 故障排除

### 连接失败
```
❌ 错误: connect ECONNREFUSED 127.0.0.1:7788
```
**解决：** 运行 `pnpm dev` 启动服务器

### 模型不存在
```
❌ 模型 "xxx" 不存在
```
**解决：** 运行 `node test_models.js` 查看可用模型

### 索引超出范围
```
❌ 动作索引 10 超出范围 (0-9)
```
**解决：** 先运行不带索引的命令查看有效范围
